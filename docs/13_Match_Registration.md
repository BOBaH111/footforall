# Match Registration Domain

## Назначение

Match Registration представляет участие пользователя в конкретном матче.

Регистрация создаётся после подачи заявки игроком и проходит полный жизненный цикл вплоть до завершения матча или отмены участия.

Регистрация является единственной точкой входа игрока в матч.

На основании регистрации выполняются:

- выбор предпочтительной игровой позиции;
- резервирование игрового места;
- оплата участия;
- формирование команд;
- check-in;
- назначение игрового слота;
- расчёт статистики;
- расчёт рейтинга.

---

# BR-REGISTRATION-001. Одна регистрация

Пользователь может иметь только одну активную регистрацию в одном матче.

Повторная регистрация запрещена.

---

# BR-REGISTRATION-002. Регистрация только в открытый матч

Регистрация возможна только если матч находится в статусе:

- REGISTRATION_OPEN

Во всех остальных статусах регистрация запрещена.

---

# BR-REGISTRATION-003. Выбор игровой позиции

При регистрации игрок обязан выбрать одну из доступных игровых позиций.

Позиция должна иметь свободные места.

Недоступные позиции не отображаются пользователю.

---

# BR-REGISTRATION-004. Резервирование места

После успешной регистрации за игроком резервируется место выбранной позиции.

До окончания времени резервирования данное место недоступно другим игрокам.

---

# BR-REGISTRATION-005. Подтверждение участия

Регистрация считается подтверждённой только после выполнения всех обязательных условий матча.

Например:

- успешная оплата;
- подтверждение организатором;
- автоматическое подтверждение.

---

# BR-REGISTRATION-006. Отмена регистрации

Игрок может отменить регистрацию до момента закрытия регистрации.

После отмены:

- место освобождается;
- регистрация получает статус CANCELLED;
- игрок исключается из формирования команд.

---

# BR-REGISTRATION-007. Закрытие регистрации

После перехода матча в состояние REGISTRATION_CLOSED:

- новые регистрации запрещены;
- изменение позиции запрещено;
- отмена регистрации выполняется только по правилам организатора.

---

# BR-REGISTRATION-008. Формирование команд

В автоматическом формировании участвуют только регистрации со статусом CONFIRMED.

Все остальные регистрации игнорируются.

---

# BR-REGISTRATION-009. Check-In

Перед началом матча игрок обязан подтвердить своё присутствие.

Только успешно прошедшие Check-In игроки допускаются к назначению игровых слотов.

---

# BR-REGISTRATION-010. Неявка

Если игрок не прошёл Check-In:

- регистрация получает статус NO_SHOW;
- игрок исключается из состава;
- слот освобождается;
- может быть приглашён игрок из листа ожидания.

---

# BR-REGISTRATION-011. История регистрации

После завершения матча регистрация становится частью постоянной истории пользователя.

Удаление регистрации запрещено.

---

# BR-REGISTRATION-012. Снимок рейтинга

При подтверждении регистрации сохраняется снимок рейтинга игрока.

Именно этот рейтинг используется при автоматическом формировании команд.

Последующие изменения общего рейтинга пользователя не влияют на уже созданный матч.

---

# BR-REGISTRATION-013. Независимость регистрации

Регистрация не хранит информацию о команде.

Регистрация не хранит информацию об игровом слоте.

Назначение команды и игрового слота выполняется после закрытия регистрации.

---

# BR-REGISTRATION-014. Основание для назначения

Только подтверждённая регистрация может быть использована для создания TeamSlotAssignment.

---

# BR-REGISTRATION-015. Смена позиции

До закрытия регистрации игрок может изменить выбранную позицию.

При изменении позиции:

* освобождается прежнее место;
* резервируется новое место;
* проверяется наличие свободного места.

Это очень жизненный сценарий: игрок записался защитником, а потом решил играть в нападении.

⸻

# BR-REGISTRATION-016. Роль в матче

Регистрация должна фиксировать роль участия.

Например:

PLAYER
REFEREE
COACH

⸻

# BR-REGISTRATION-017

После начала регистрации место резервируется

на ограниченное время.

⸻

# BR-REGISTRATION-018

Регистрация поддерживает список предпочтений игровых позиций.

Для MVP используется только requestedPosition.

В последующих версиях requestedPosition станет первым элементом PositionPreferences.
