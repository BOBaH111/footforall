# Assignment Engine

## Назначение

Assignment Engine выполняет окончательное назначение игроков на игровые позиции после формирования сбалансированных команд.

Основная задача Engine — сформировать максимально честный состав команд, учитывая:

- предпочтения игроков;
- силу игроков на каждой позиции;
- баланс команд;
- правила конкретного формата матча.

Assignment Engine является последним этапом подготовки матча перед его началом.

После завершения работы Engine создаются TeamSlotAssignment.

# BR-ASSIGN-001. Независимость этапов

Формирование команд и назначение игровых позиций являются независимыми этапами.

Formation Engine определяет принадлежность игрока к одной из команд.

Assignment Engine определяет игровую позицию игрока внутри уже сформированной команды.

---

# BR-ASSIGN-002. Работа внутри команды

Assignment Engine работает отдельно для каждой команды.

Игроки разных команд между собой не сравниваются.

Баланс между командами считается завершённым после окончания Formation Engine.

---

# BR-ASSIGN-003. Приоритеты игровых позиций

Каждый игрок перед регистрацией обязан указать полный порядок предпочтений игровых позиций.

Например:

1. Forward
2. Midfielder
3. Defender
4. Goalkeeper

Данные предпочтения используются исключительно Assignment Engine.

---

# BR-ASSIGN-004. Рейтинг каждой позиции

Для каждой игровой позиции существует собственный рейтинг игрока.

Например:

Forward — 94

Midfielder — 81

Defender — 72

Goalkeeper — 28

При расчёте силы состава используется рейтинг игрока именно на назначенной позиции.

---

# BR-ASSIGN-005. Первичная попытка назначения

Engine всегда сначала пытается назначить игрока на наиболее предпочтительную позицию.

Переход к следующему приоритету допускается только при невозможности построить корректный состав.

---

# BR-ASSIGN-006. Баланс состава

Если существует несколько допустимых вариантов назначения игроков, Engine выбирает вариант с минимальной разницей общей силы команд.

---

# BR-ASSIGN-007. Объяснимость решений

Каждое назначение должно быть объяснимым.

Для любого TeamSlotAssignment система должна иметь возможность определить:

- почему была выбрана именно эта позиция;
- какие альтернативы рассматривались;
- почему они были отклонены.

Это обеспечивает прозрачность работы платформы.

---

# BR-ASSIGN-008. Право отказаться

Если игрок получил позицию ниже установленного уровня предпочтений, система предоставляет ему право отказаться от участия с полным возвратом денежных средств.

Порог определяется бизнес-настройками платформы.

Для MVP предполагается использование второго приоритета.

Если игрок получил третью или более низкую позицию, ему предоставляется возможность отказаться от участия.

---

# BR-ASSIGN-009. Подтверждение назначения

После завершения расчёта Assignment Engine система уведомляет только тех игроков, которые получили альтернативную позицию.

Игроки, получившие первую или вторую предпочтительную позицию, подтверждение не проходят.

Игроки, получившие третью или более низкую позицию, должны:

- подтвердить назначение;
- либо отказаться от участия.

При отказе игрок исключается из матча, а его место становится доступным для повторного назначения.

---

# BR-ASSIGN-010. Создание TeamSlotAssignment

После завершения подтверждений создаются TeamSlotAssignment.

Каждый TeamSlotAssignment содержит:

- MatchRegistration;
- MatchTeam;
- TeamSlot;
- назначенную игровую позицию.

Именно TeamSlotAssignment становится источником данных для Match Execution Engine.

---

# Долгосрочное развитие

В будущих версиях платформы Assignment Engine будет использовать систему оценки вариантов назначения (Assignment Score).

Каждая возможная конфигурация состава будет оцениваться по совокупности факторов:

- предпочтения игроков;
- сила игроков на выбранных позициях;
- баланс команд;
- дополнительные бизнес-правила.

Engine будет выбирать конфигурацию с максимальной итоговой оценкой.

В MVP используется детерминированный алгоритм назначения без расчёта Assignment Score.
