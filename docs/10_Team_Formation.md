# Team Formation Domain

## Назначение

Team Formation отвечает за создание составов команд внутри конкретного матча.

Команда является временной сущностью и существует только в рамках одного матча.

История состава команды является частью истории матча.

---

# BR-TEAM-001. Формирование команд

Команды создаются только после завершения регистрации игроков.

Условия:

- регистрация игроков закрыта;
- все игроки подтверждены;
- известны игровые позиции участников.

---

# BR-TEAM-002. Автоматическое формирование

Для товарищеских матчей используется автоматическое распределение.

Алгоритм учитывает:

- выбранное амплуа игрока;
- уровень игрока;
- рейтинг;
- опыт;
- баланс команд.

---

# BR-TEAM-003. Амплуа игрока

Амплуа выбирается игроком при регистрации на конкретный матч.

Амплуа не является постоянным свойством пользователя.

Пример:

Игрок может выступать:

Матч 1:
DEFENDER

Матч 2:
FORWARD

---

# BR-TEAM-004. Формирование по позициям

Алгоритм должен обеспечить:

- необходимое количество игроков каждой позиции;
- наличие вратаря;
- баланс обороны и атаки.

---

# BR-TEAM-005. Ручное формирование

Для турниров составы команд могут быть определены организатором заранее.

Функция добавляется на следующих этапах.

---

# BR-TEAM-006. Неизменяемость состава

После начала матча:

- состав команд фиксируется;
- изменение возможно только через специальные процедуры;
- статистика привязывается к зафиксированному составу.

---

# BR-TEAM-008. Баланс игровых линий

При автоматическом формировании команд система не рассматривает всех игроков как единую группу.

Игроки разделяются на две функциональные группы:

## Defensive Group

Включает:

- GOALKEEPER
- DEFENDER


## Offensive Group

Включает:

- MIDFIELDER
- FORWARD


Балансировка команд выполняется отдельно для каждой группы.

Цель:

Минимизировать разницу между командами как по оборонительной, так и по атакующей составляющей.

---

# BR-TEAM-009. Расчёт силы команды

Сила команды рассчитывается отдельно:

Defensive Power:

GOALKEEPER rating +
DEFENDER ratings


Offensive Power:

MIDFIELDER ratings +
FORWARD ratings

---

# BR-TEAM-010. Сохранение структуры состава

При автоматическом формировании команд система обязана сохранять структуру игровых позиций матча.

Алгоритм формирования не может компенсировать отсутствие одной позиции только высоким рейтингом игроков другой позиции.

---

## Требования

После формирования команд каждая команда должна получить:

- необходимое количество игроков каждой позиции;
- обязательного вратаря (если предусмотрено форматом);
- соответствующее количество игроков оборонительной и атакующей группы.

---

## Пример

Матч 6×6:

Основной состав:

- GOALKEEPER ×1
- DEFENDER ×2
- MIDFIELDER ×1
- FORWARD ×2


Результат формирования:

Team A:

- GOALKEEPER ×1
- DEFENDER ×2
- MIDFIELDER ×1
- FORWARD ×2


Team B:

- GOALKEEPER ×1
- DEFENDER ×2
- MIDFIELDER ×1
- FORWARD ×2


---

## Ограничения

Если невозможно создать две полноценные команды с заданной структурой:

- матч не переходит к формированию команд;
- система сообщает о невозможности формирования;
- организатор получает возможность изменить параметры матча или открыть дополнительные позиции.

---

## Приоритеты алгоритма

При формировании команд применяются следующие приоритеты:

1. Сохранение структуры состава.
2. Баланс оборонительной группы.
3. Баланс атакующей группы.
4. Минимизация общей разницы рейтинга команд.

Учитываются:
- Defensive Power
- Offensive Power

---

# BR-TEAM-011. Проверка соответствия регистрации формату матча

Перед формированием команд система должна проверить:

- количество игроков каждой позиции;
- наличие обязательных позиций;
- возможность создания двух полноценных команд.

Если структура игроков не соответствует формату:

- формирование команд запрещается;
- возвращается причина невозможности.
