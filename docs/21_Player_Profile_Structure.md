# 21. Player Profile Structure

## Purpose

Player Profile является агрегатом верхнего уровня Player Domain.

Сам агрегат практически не содержит бизнес-логики.

Большая часть поведения распределена между специализированными Value Objects и Domain Components.

Такой подход позволяет развивать различные части цифрового профиля независимо друг от друга.

---

# Aggregate

```
Player Profile
```

Player Profile является единственной точкой изменения цифровой футбольной личности игрока.

Никакая другая сущность системы не может изменять его состояние напрямую.

---

# Internal Structure

```
Player Profile

├── Identity
├── Life Stage
├── Overall Rating
├── Position Ratings
├── Match Statistics
├── Reputation
├── Achievements
├── Evolution History
└── Preferences
```

Каждый компонент отвечает только за собственную предметную область.

---

# Identity

Отвечает за неизменяемую идентификацию профиля.

Содержит:

- Player Profile Id
- User Id
- Created At
- Status

Identity никогда не изменяется после создания.

---

# Life Stage

Определяет возрастную категорию игрока.

Возможные значения:

- U12
- U16
- Adult
- Veterans

Переходы:

```
U12
↓

U16
↓

Adult
↓

Veterans
```

Переходы U12 → U16 и U16 → Adult происходят автоматически.

Переход Adult → Veterans инициируется пользователем и является необратимым.

---

# Overall Rating

Отражает общий игровой уровень.

Является производным показателем.

Изменяется исключительно Evolution Engine.

---

# Position Ratings

Для каждой игровой позиции хранится отдельный рейтинг.

Все позиции доступны игроку.

Система не ограничивает развитие.

---

# Match Statistics

Содержит исключительно накопительную статистику.

Например:

- Matches
- Minutes
- Wins
- Draws
- Losses
- Goals
- Assists
- Saves
- Cards
- MVP

Структура может расширяться.

---

# Reputation

Отражает доверие системы к игроку.

Не зависит напрямую от игрового рейтинга.

Основные источники изменения:

- дисциплина;
- подтверждение участия;
- завершённые матчи;
- неявки;
- отмены;
- оценки официальных лиц.

---

# Achievements

Содержит все полученные достижения.

Каждое достижение неизменно после получения.

---

# Evolution History

Хранит журнал изменений Player Profile.

Каждая запись содержит:

- источник изменения;
- предыдущие значения;
- новые значения;
- дату;
- Engine, выполнивший изменение.

История никогда не изменяется.

---

# Preferences

Содержит пользовательские настройки.

Например:

- предпочитаемый номер;
- ведущая нога;
- предпочитаемый стиль уведомлений;
- другие параметры, не влияющие на расчёт рейтингов.

Preferences не участвует в Evolution Engine.

---

# Snapshot Principle

Перед запуском любого Engine создаётся Snapshot Player Profile.

Все вычисления производятся только относительно Snapshot.

После завершения Engine формируется новая версия Player Profile.

---

# Design Principles

Player Profile хранит только текущее состояние.

История изменений хранится отдельно.

Все вычисляемые показатели изменяются только специализированными Engine.

Player Profile никогда не изменяется вручную.
