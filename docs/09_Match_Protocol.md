# Match Protocol Domain

## Назначение

Match Protocol является официальным итоговым документом матча.

Протокол создаётся после завершения матча на основании зафиксированных игровых событий.

Протокол является источником данных для:

- статистики игроков;
- рейтингов;
- достижений;
- финансовых операций;
- истории матчей пользователя.

---

# BR-PROTOCOL-001. Создание протокола

## Описание

Протокол может быть создан только после завершения матча.

## Условия

- матч находится в статусе FINISHED;
- все игровые события сохранены;
- судья имеет право завершить протокол.

## Результат

Создаётся MatchProtocol.

Статус протокола:

Draft.

# BR-PROTOCOL-002. Неизменяемость протокола

После подтверждения протокола:

- результат матча нельзя изменить;
- состав участников нельзя изменить;
- игровые события нельзя удалить;
- рейтинг может быть пересчитан только через специальную процедуру исправления.

---

# BR-PROTOCOL-003. Формирование счёта

Счёт матча рассчитывается только из событий типа GOAL.

Каждое событие GOAL имеет параметр:

GoalType:

- NORMAL;
- OWN_GOAL;
- PENALTY.

Все виды голов учитываются в общем счёте матча.

Пример:

Команда А:

- GOAL × 3

Команда B:

- GOAL × 2

Итог:

3:2

---

# BR-PROTOCOL-004. Статистика игроков

После формирования протокола рассчитывается:

Игрок:

- количество матчей;
- минуты на поле;
- голы;
- передачи;
- жёлтые карточки;
- красные карточки;
- сухие матчи (для вратарей и защитников);
- участие в победных матчах;
- оценки после матча.

---

# BR-PROTOCOL-005. Лучший игрок матча

MVP рассчитывается после завершения протокола.

Алгоритм может учитывать:

- голы;
- передачи;
- позицию игрока;
- действия в защите;
- оценку судьи;
- результат команды;
- оценку игроков.

# BR-PROTOCOL-007. Связь с рейтингами

После подтверждения протокола запускаются процессы:

- обновление рейтинга игроков;
- обновление рейтинга судьи;
- обновление рейтинга площадки;
- начисление достижений.

---

# BR-PROTOCOL-008. Связь с финансами

После подтверждения протокола:

- матч считается выполненным;
- создаются финансовые операции;
- становится возможна выплата судье;
- фиксируется доход площадки;
- рассчитывается комиссия платформы.
