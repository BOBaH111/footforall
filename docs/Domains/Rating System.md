# 08. Rating System

## Назначение

Rating System — единая система расчёта цифровой репутации участников экосистемы Footforall.

Рейтинг используется для:

- оценки качества участников;
- формирования равных матчей;
- построения лидербордов;
- рекомендаций игроков;
- оценки качества судей;
- оценки качества площадок;
- формирования доверия внутри платформы.

Каждая роль имеет собственную модель расчёта, однако все модели работают по единым архитектурным принципам.

---

# Принципы

## Объективность

Рейтинг рассчитывается только на основании подтверждённых системой событий.

Ручное изменение рейтингов запрещено.

---

## Автоматический пересчёт

Каждый завершённый матч автоматически инициирует пересчёт рейтингов всех участвующих сущностей.

---

## Независимость моделей

Каждая роль имеет собственную модель расчёта.

Например:

- Player Rating
- Referee Rating
- Venue Rating

Изменение формулы одной роли не должно влиять на остальные.

---

## История

Каждое изменение рейтинга сохраняется.

Система должна позволять восстановить рейтинг на любую дату.

---

## Настраиваемость

Все коэффициенты должны храниться в конфигурации.

Изменение коэффициентов не требует изменения исходного кода.

---

## Версионирование

Каждая версия алгоритма получает собственный номер.

Например

Rating Algorithm v1

↓

Rating Algorithm v2

↓

Rating Algorithm v3

Это позволит менять модель расчёта без потери истории.

---

# Общая модель

Каждый рейтинг рассчитывается движком.

Например

PlayerRatingEngine

RefereeRatingEngine

VenueRatingEngine

Каждый движок реализует единый контракт.

interface RatingEngine<TEvidence, TResult>

Каждый движок принимает объективные игровые данные.

Каждый движок возвращает новый рейтинг.

# Будущие возможности

Планируется поддержка:

- различных рейтинговых моделей;
- сезонных рейтингов;
- турнирных коэффициентов;
- коэффициентов сложности;
- региональных рейтингов;
- мирового рейтинга;
- рейтингов по типам покрытия;
- рейтингов по формату матча;
- рейтингов по возрастным категориям.

Архитектура должна позволять добавление новых моделей без изменения существующих.
